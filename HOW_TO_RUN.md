# aeris OS — Как запустить

Полнофункциональный интерфейс операционной системы в стиле **aeris Aero**
для Unity 2D. Главное меню, рабочий стол с иконками, перетаскиваемые
окна приложений, системная панель с часами, toast-уведомления,
экран загрузки с прогрессбаром, синтезированные beep-звуки.

> **Главная фишка:** ничего вручную в Unity Editor собирать не нужно.
> Открыли сцену → нажали Play → всё работает. Весь UI и все ресурсы
> (текстуры, звуки) генерируются скриптами в рантайме.

---

## 1. Требования

| Что | Версия |
|-----|--------|
| **Unity Hub** | любая актуальная |
| **Unity Editor** | **`6000.4.4f1`** (точно эта — проект сохранён в этом формате) |
| **ОС** | Windows / macOS / Linux |
| **Платформа сборки** | Standalone (PC, Mac, Linux) |
| **Сторонние ассеты / пакеты** | не требуются |

Если Unity Hub попросит другую версию редактора — установите именно
`6000.4.4f1`. Можно открыть и в более новой версии Unity 6, но тогда
Hub предложит «обновить проект» — соглашайтесь, ошибок быть не должно.

## 2. Открытие проекта (~3 минуты)

1. Запустите **Unity Hub**.
2. Нажмите **Add → Add project from disk**.
3. Выберите папку **`final_gamedev/Pet`** (именно `Pet`, не корень `final_gamedev`).
4. Подождите, пока Unity импортирует пакеты и скомпилирует скрипты
   (первый раз — 2–5 минут, дальше быстро).
5. В **Console** не должно быть красных ошибок. Жёлтые warnings допустимы.

---

## 3. Запуск (1 действие)

1. В **Project**-панели откройте **`Assets/Scenes/SampleScene.unity`**.
2. Нажмите **▶ Play** в верхней панели редактора.

Всё. Сцена пустая, но скрипт `GameBootstrap` помечен атрибутом
`[RuntimeInitializeOnLoadMethod]` — он сам создаёт `Canvas`,
`EventSystem`, `AudioManager` и собирает весь UI на старте сцены.

> Если в Build-настройках сцены нет — добавьте через
> **File → Build Settings → Add Open Scenes**.

---

## 4. Что делать в игре

### Главное меню
- **Start** — экран загрузки → рабочий стол.
- **Settings** — открывает окно «Settings» поверх рабочего стола.
- **Exit** — закрывает приложение (в Editor — останавливает Play).

### Рабочий стол
- Слева 5 иконок: **My Computer, Notes, Music, Browser, Settings**.
- **Двойной клик** по иконке открывает соответствующее окно.
- Окно можно **перетащить** за заголовок.
- Кнопка **X** закрывает окно (с fade-out).
- Кнопка **_** — заглушка минимизации.

### Системная панель (нижний таскбар)
- Слева — зелёный «Start»-орб.
- В центре — виджет погоды (плейсхолдер `Sunny +24°C`).
- Справа — 3 цветных индикатора + **часы** (тикают каждый кадр от `DateTime.Now`) и текущая дата.

### Уведомления
- При открытии любого окна в правом верхнем углу появляется toast:
  `"<App> opened"` со звуком notify.

### Окно Settings
- **Master Volume** — ползунок громкости звуков.
- **Test Click / Test Notify** — проиграть beep.
- **Back to Menu** — вернуться в главное меню (с fade-переходом).

## 5. Сборка standalone-билда

1. **File → Build Profiles** (или **Build Settings**).
2. В списке Scenes In Build должна быть `SampleScene` (если нет — `Add Open Scenes`).
3. Выберите Platform: **Windows / Mac / Linux**.
4. Нажмите **Build** → выберите папку для `.exe`.
5. Запустите получившийся файл — поведение идентично Editor-режиму.

## 9. FAQ / Частые проблемы

**Unity Hub просит другую версию редактора.**
Установите Unity `6000.4.4f1` через Hub → Installs → Install Editor →
Archive (или нужная вкладка). Или согласитесь открыть в более новой
версии 6.x — проект совместим, Unity предложит миграцию.

**На сцене ничего не появляется при Play.**
Убедитесь, что открыта `SampleScene.unity` (а не пустая «Untitled»).
Проверьте Console на ошибки компиляции — если есть красные сообщения,
скрипты не скомпилировались и `GameBootstrap` не запустился.

**Нет звука.**
Откройте Settings → ползунок Master Volume не на нуле. Проверьте, что
системный микшер не заглушил Unity-приложение.

**Кнопки не реагируют на клики.**
Это означает, что не создалась `EventSystem`. Перезапустите Play.
Если повторяется — добавьте вручную: иерархия → Create → UI → Event System.

**`InvalidOperationException: You are trying to read Input using the UnityEngine.Input class…`**
Проект настроен на новый Input System, а старый модуль ввода читает
устаревший API. В скрипте `GameBootstrap` это уже учтено через
`#if ENABLE_INPUT_SYSTEM` — он подключает `InputSystemUIInputModule`
автоматически. Если ошибка всё равно есть: **Edit → Project Settings →
Player → Other Settings → Active Input Handling = `Input System Package
(New)`** или `Both`, перезапустите Unity, нажмите Play.

**Окна выходят за пределы экрана.**
Это by design (как в реальной ОС). Перетащите за заголовок обратно.

**Часы показывают неправильное время.**
Они берут `DateTime.Now` — поправьте время системы.

## 10. Что дальше

Архитектура полностью кодовая, поэтому расширять легко:

- **Новое приложение:** добавьте значение в `enum AppType`, кейс в
  `WindowManager.FillBody()`, иконку в `DesktopBuilder.Build()`.
- **Новый звук:** добавьте метод `Synthesize…()` в `AudioManager` и поле-клип.
- **Сохранение настроек громкости:** `PlayerPrefs.SetFloat("vol", ...)` в `Settings`-окне.
- **Реальная погода:** замените текст `"Sunny +24°C"` на ответ от API через `UnityWebRequest`.
